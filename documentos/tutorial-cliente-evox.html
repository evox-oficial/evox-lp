Guia detalhado para alterar o conteúdo do site diretamente pelo GitHub, sem precisar de programa adicional.

Este documento explica passo a passo como criar sua conta no GitHub, acessar o repositório e editar cada tipo de conteúdo do site EVOX.

GitHub Banners Modelos Fotos Textos Formulário Tally Páginas Legais

1. O que você pode alterar

✅ Alterações seguras

  • Banners da home (texto, imagem, cor, botões)
  • Modelos visíveis na vitrine da home
  • Fotos de capa e galeria dos modelos
  • Imagens dos banners (upload de arquivo)
  • Link do formulário Tally
  • Páginas legais (termos, privacidade, garantia)

⚠️ Alterações com atenção

  • Textos e especificações de DOHA e VIENNA
  • Páginas dos modelos futuros (modelo-03 a 09)
  • Textos institucionais do index.html
  • ID do formulário Tally no index.html

2. Criando sua conta no GitHub

O GitHub é onde os arquivos do site ficam guardados. Você precisa de uma conta gratuita para poder editar.

  1. Abra o site do GitHub No navegador, acesse: https://github.com
  2. Clique em "Sign up" — botão no canto superior direito Será aberta a tela de cadastro.
  3. Preencha seu e-mail e clique em "Continue" Use um e-mail que você acessa com frequência.
  4. Crie uma senha e clique em "Continue" Mínimo de 8 caracteres, com letras e números.
  5. Escolha um nome de usuário e clique em "Continue" Exemplo: evox-marketing ou evox-cliente. Esse nome vai aparecer no histórico de edições.
  6. Resolva o captcha (quebra-cabeça visual) e clique em "Create account"
  7. Verifique seu e-mail O GitHub vai enviar um código de 6 dígitos. Copie e cole na tela do GitHub para ativar a conta.
  8. Pule as perguntas opcionais clicando em "Skip personalization" Não é necessário preencher preferências.
Conta criada! O GitHub é gratuito e não precisa de cartão de crédito.

3. Acessando o repositório do site EVOX

Depois de criar a conta, você precisa ser adicionado como colaborador pelo desenvolvedor. Envie seu nome de usuário do GitHub para ele antes de continuar.

  1. Faça login no GitHub Acesse https://github.com, clique em "Sign in" e informe e-mail e senha.
  2. Acesse o repositório do site Digite o endereço abaixo na barra do navegador:
    https://github.com/renan-barreto4/evox
  3. Você verá a lista de pastas e arquivos do site É a mesma estrutura do site: js/, modelos/, images/, img-slider/ etc.
  4. Para entrar em uma pasta, clique no nome dela Para voltar, clique no nome do repositório no topo ou no botão de voltar do navegador.
Se ao clicar no lápis de edição aparecer a mensagem "You must be on a branch to edit a file", avise o desenvolvedor — ele precisa te adicionar como colaborador com permissão de escrita.

4. Como editar qualquer arquivo de texto pelo GitHub

Este processo se repete para todos os arquivos editáveis: banners, modelos, textos, páginas legais etc.

  1. Navegue até o arquivo desejado Clique nas pastas para entrar nelas até chegar ao arquivo.
  2. Clique no ícone de lápis ✏️ ("Edit this file") Fica no canto superior direito da área de exibição do arquivo, junto com outros ícones.
  3. O editor de texto vai abrir Você pode clicar em qualquer parte do texto para posicionar o cursor e digitar normalmente.
  4. Faça apenas a alteração necessária Não apague chaves { }, colchetes [ ], vírgulas ou aspas que não fazem parte do texto visível.
  5. Role a página até o final — seção "Commit changes" Você verá um campo de mensagem.
  6. Escreva uma descrição curta do que mudou Exemplos: "Atualiza banner Dia das Mães", "Ativa modelo CLOUD na home", "Corrige texto da privacidade".
  7. Clique no botão verde "Commit changes" A alteração é salva. O site atualiza automaticamente em poucos segundos.
Dica de segurança: antes de editar, copie o conteúdo atual para um bloco de notas. Se algo der errado, você consegue restaurar.

5. Editando os banners da home

Todos os banners são controlados por um único arquivo. Abra-o pelo GitHub assim:

js / campaign-banners-config.js

Use o lápis ✏️ para editar (siga os passos da Seção 4). O arquivo tem 4 banners. Cada banner começa com { e termina com },.

5.1 — Ativar ou desativar um banner

Cada banner tem a linha ativo: true ou ativo: false. Altere apenas esse valor.

// Para MOSTRAR o banner: ativo: true, // Para ESCONDER o banner: ativo: false,

5.2 — Trocar o texto do título

O título aceita uma quebra de linha com <br> e destaque em cor especial com a tag <span class="highlight">.

// Título simples (uma linha): titulo: 'PROMOÇÃO DE LANÇAMENTO', // Título com destaque colorido e quebra de linha: titulo: 'PROMOÇÃO DE <br><span class="highlight">LANÇAMENTO</span>',
A palavra dentro de <span class="highlight">...</span> vai aparecer na cor especial definida em corEspecial.

5.3 — Trocar o subtítulo

subtitulo: 'Novo texto explicativo do banner aqui.',

Para esconder o subtítulo completamente:

esconderSubtitulo: true,

5.4 — Mudar a cor especial do banner

Só existem três valores permitidos:

corEspecial: 'azul', // destaques em azul corEspecial: 'verde', // destaques em verde corEspecial: 'amarelo', // destaques em dourado
Não use outras palavras. Se escrever algo diferente, a cor especial não vai funcionar.

5.5 — Trocar a imagem de fundo do banner

As imagens dos banners ficam na pasta img-slider/. Os arquivos atuais são:
Desktop: 1.jpeg, 2.jpeg, 3.jpeg, 4.jpeg, 5.jpeg
Mobile: 1-mobile.jpeg, 2-mobile.jpeg, 3-mobile.jpeg, 4-mobile.jpeg

Para trocar a imagem de um banner:

  1. Faça o upload da nova imagem na pasta img-slider/ (veja Seção 7 para como enviar imagens).
  2. Abra o campaign-banners-config.js e atualize o caminho:
imagemDesktop: 'img-slider/nova-foto.jpeg', imagemMobile: 'img-slider/nova-foto-mobile.jpeg',

5.6 — Editar os botões do banner

Cada banner tem dois botões: botaoPrincipal e botaoSecundario.

botaoPrincipal: { mostrar: true, // true = mostrar, false = esconder texto: 'FALAR COM ESPECIALISTA', link: 'https://wa.me/5511999999999', icone: 'fa-arrow-right' // não alterar },
Para esconder um botão, troque mostrar: true para mostrar: false. Não apague o bloco inteiro.

5.7 — Editar as especificações técnicas do banner

Alguns banners mostram caixinhas de especificação (autonomia, potência etc.). Cada caixinha é um item na lista especificacoes:

especificacoes: [ { icone: 'fa-solid fa-route', rotulo: 'ATE', valor: '55 <span>KM</span>', textoApoio: 'DE AUTONOMIA' }, { icone: 'fa-solid fa-bolt', rotulo: '', valor: '1000<span>W</span>', textoApoio: 'POTENCIA NOMINAL' }, ]

Para esconder todas as especificações de um banner:

mostrarEspecificacoes: false,

6. Ativar ou editar modelos na vitrine da home

A vitrine de modelos é controlada pelo arquivo:

js / models-config.js

Abra-o pelo GitHub com o lápis ✏️ (Seção 4). Cada modelo é um bloco entre { e },.

6.1 — Mostrar ou esconder um modelo

// Para mostrar o modelo na vitrine: visible: true, // Para esconder (modelo futuro ainda não lançado): visible: false,

6.2 — Editar os dados de um modelo (ex.: DOHA)

Localize o bloco do modelo desejado e edite os campos abaixo:

{ visible: true, name: 'DOHA', // Nome exibido no card folder: 'doha', // Nome da pasta em modelos/ description: 'Estrutura robusta, presença marcante...', link: 'modelos/doha/doha.html', // não alterar accentColor: '#e5b80b', // cor do destaque do card (hex) accentTextColor: '#000000', // cor do texto sobre o destaque buttonLabel: 'Explorar Doha', // texto do botão do card colors: ['Cinza', 'Verde', 'Azul'] // opções de cor listadas },

6.3 — Ativar um modelo futuro (ex.: modelo-03)

  1. Suba as fotos do novo modelo na pasta modelos/modelo-03/img/ (veja Seção 7). Inclua uma foto chamada capa.jpg.
  2. Abra o models-config.js e localize o bloco NOVO MODELO 3.
  3. Altere os campos:
    visible: false → visible: true
    name, description, buttonLabel e colors com os dados reais do modelo.
  4. Faça o commit com uma mensagem como "Ativa modelo CLOUD na home".

7. Enviando imagens pelo GitHub

Para trocar fotos de banners, capas ou galerias, você faz upload diretamente pelo GitHub — sem precisar de programa extra.

  1. Acesse o repositório e entre na pasta de destino
    Banners: img-slider/  |  Capa DOHA: modelos/doha/img/  |  Capa VIENNA: modelos/vienna/img/
  2. Clique no botão "Add file" (acima da lista de arquivos) → "Upload files"
  3. Arraste as imagens para a área pontilhada ou clique em "choose your files"
    Formatos aceitos: .jpg .jpeg .png .webp
  4. Aguarde o carregamento (barra de progresso)
    Arquivos grandes podem demorar alguns segundos.
  5. Role até "Commit changes", escreva uma mensagem e clique no botão verde
    Ex.: "Adiciona foto nova do DOHA", "Troca banner desktop campanha junho".
Dica — capa da home: a imagem de capa de cada modelo deve se chamar exatamente capa.jpg (ou .jpeg / .png / .webp). Enviando uma nova imagem com esse nome, ela substitui a anterior automaticamente.
Tamanho máximo recomendado: 800 KB por imagem. Imagens maiores deixam o site lento. Reduza o tamanho antes de enviar em: squoosh.app ou tinypng.com (gratuitos).

8. Editar textos e especificações de DOHA e VIENNA

Cada modelo tem sua própria página HTML. Abra pelo GitHub com o lápis ✏️ (Seção 4).

DOHA

modelos / doha / doha.html

VIENNA

modelos / vienna / vienna.html

O que você pode editar nessas páginas:

Título do modelo Texto grande no topo da página. Procure pela tag <h1>.
SKU / código Código do produto exibido abaixo do título. Procure por algo como DOHA-2024.
Texto de apresentação Parágrafo descritivo do modelo. Está dentro de uma tag <p>.
Cores disponíveis Lista com os nomes das cores oferecidas. Procure pela lista de cores na página.
Tabelas de specs Dados como potência, autonomia, freios, peso. Altere apenas os valores numéricos e textuais — não apague as tags HTML.
Botão de interesse Link e texto do botão de contato no final da página. Procure por <a href= próximo ao final do arquivo.
Atenção ao editar HTML: troque apenas o texto que fica entre as tags. Não apague os símbolos < e > nem os nomes das tags.
Exemplo seguro: <h1>DOHA PRO</h1> — altere apenas o que está entre as tags.

9. Trocar o formulário de contato (Tally)

O formulário da home usa a plataforma Tally. Para apontar para um novo formulário, você só precisa trocar o ID.

  1. Crie ou abra o formulário no Tally Acesse https://tally.so, faça login e abra o formulário desejado.
  2. Copie o ID do formulário O link do formulário publicado tem este formato: https://tally.so/r/ABC123. O ID é a parte final — no exemplo, ABC123.
  3. Abra o arquivo index.html pelo GitHub com o lápis ✏️
    index.html
  4. Use Ctrl+F para localizar TALLY_FORM_ID Você vai encontrar uma linha parecida com esta:
    const TALLY_FORM_ID = 'ABC123';
  5. Substitua apenas o valor entre aspas pelo novo ID Não altere o restante da linha.
  6. Faça o commit com a mensagem "Atualiza ID do formulário Tally".

10. Textos fixos da home (index.html)

Além dos banners, alguns textos da home estão escritos diretamente no arquivo principal. Abra pelo GitHub com o lápis ✏️ e use Ctrl+F para localizar o trecho.

index.html
Seção de contato Procure pelo texto atual da seção e substitua. Não altere as tags HTML ao redor.
"Ainda tem dúvidas?" Procure pela frase exata para localizar e editar o bloco.
Botões fixos da home Procure pelo texto do botão atual para localizar a linha e alterar o texto ou o link (href).
Textos institucionais Textos de seções como tecnologia, missão etc. Localize pela frase e edite apenas o conteúdo entre as tags.
Recomendação: para campanhas, prefira sempre editar o js/campaign-banners-config.js (Seção 5). O index.html é mais complexo — edite com cuidado.

11. Páginas legais e institucionais

Essas páginas já existem e podem receber os textos finais. Abra cada uma pelo GitHub com o lápis ✏️ (Seção 4).

Termos de Uso

termos.html

Privacidade

privacidade.html

Garantia

garantia.html
  1. Abra o arquivo desejado no GitHub e clique no lápis ✏️.
  2. Localize a área de conteúdo — procure pelo texto de exemplo atual usando Ctrl+F.
  3. Substitua o texto de exemplo pelo texto jurídico oficial. Mantenha as tags <p>, <h2>, <ul> para formatar corretamente.
  4. Faça o commit com a mensagem "Atualiza texto da política de privacidade" (ou similar).

12. Ordem recomendada para uma atualização completa

Siga esta sequência para não esquecer nenhum passo em uma troca de campanha.

  1. Prepare as imagens novas no computador. Reduza o tamanho em squoosh.app se necessário.
  2. Envie as imagens novas pelo GitHub para as pastas corretas (Seção 7).
  3. Edite os banners em js/campaign-banners-config.js — atualize título, subtítulo, cor especial e caminho das imagens (Seção 5).
  4. Se algum modelo mudou, edite js/models-config.js para ativar/desativar ou atualizar dados (Seção 6).
  5. Se os textos do DOHA ou VIENNA mudaram, edite o HTML de cada um (Seção 8).
  6. Se o formulário mudou, atualize o TALLY_FORM_ID no index.html (Seção 9).
  7. Abra o site no navegador e verifique se tudo está correto. O site atualiza em até 1 minuto após o commit.
Histórico de alterações: cada commit fica registrado no GitHub com data, hora e quem fez. Se algo der errado, o desenvolvedor consegue reverter para qualquer versão anterior — nada é perdido permanentemente.
